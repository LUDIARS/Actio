# Calendar（カレンダー）設計仕様

## 概要

個人の予定管理と Google Calendar との双方向同期を提供するモジュール。Google 認証なしでも手動で予定を登録できる。M2（データ統合）の役割を兼ね、各データソースを統合する。

## ユースケース

### UC-CAL-01: 個人予定の手動登録

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. タイトル・曜日・コマ（または開始/終了時刻）を入力する
  2. 公開/非公開を選択する
  3. 個人予定として登録する
- **制約**: 同一ユーザーの同一曜日・同一コマに重複登録は不可

### UC-CAL-02: 予定の編集・削除

- **アクター**: 予定の所有者
- **主フロー**:
  1. 対象の予定を選択する
  2. 内容を変更または削除する
  3. Google Calendar 連携中の場合、外部にも変更を反映する

### UC-CAL-03: Google Calendar 同期

- **アクター**: Google 連携済みユーザー
- **事前条件**: Google OAuth でカレンダースコープが認可済み
- **主フロー**:
  1. Google Calendar から予定を取得する
  2. ローカル予定と差分を検出する
  3. 双方向に同期する（作成・更新・削除）
- **同期ログ**: すべての同期操作を記録する

### UC-CAL-04: 統合スロットの計算（M2機能）

- **アクター**: システム内部
- **主フロー**:
  1. M1の確定授業予定を取得する
  2. 個人カレンダーの予定を取得する
  3. Google Calendar の予定を統合する
  4. M4の予約データをフィードバックする
  5. 各コマの状態（free / class / personal / event / reserved）を算出する
- **出力**: UnifiedSlot マトリクス（7曜日 × 11コマ）

### UC-CAL-05: 空き状況の他者への公開

- **アクター**: グループメンバー
- **主フロー**:
  1. 他のメンバーの空き状況を照会する
  2. 「予定あり」としてのみ表示する（詳細は非公開）

## プライバシー規則

| データ種別 | 公開範囲 | 他者への公開内容 |
|-----------|---------|----------------|
| 授業予定 | 専攻学科名のみ | 「情報工学・2限」のように科目名は非公開 |
| 個人カレンダー | 本人のみ | 「予定あり」としてのみ表示 |
| 学校行事 | 全ユーザー | 行事名・日時を公開 |
| 予約情報 | 全ユーザー | タイトル・教室・参加者を公開 |

## 統合スロットモデル

| 状態 | 説明 | ソース |
|------|------|--------|
| free | 空き | — |
| class | 授業 | M1 |
| personal | 個人予定 | 手動入力 / Google Calendar |
| event | 学校行事 | Google Calendar / グループイベント |
| reserved | 予約済み | M4 |

## キャッシュ戦略

- M1データ: 学期単位でキャッシュ、変更時に無効化
- Google Calendar: 15分間隔でポーリング、差分同期
- 個人カレンダー: ユーザーリクエスト時に取得（10分TTL）
- M4フィードバック: イベントドリブンで即時反映
