# M1: 時間割自動生成 設計仕様

## 概要

動的計画法（DP）と制約充足問題（CSP）を組み合わせ、カリキュラムを時間割に自動配置するサブモジュール。2つの配置モードと入れ替え候補のハイライト機能を持つ。

## ユースケース

### UC-M1S-01: 時間割の自動生成

- **アクター**: 教務管理者
- **事前条件**: カリキュラム・講師・教室・出講可能スロットが登録済み
- **主フロー**:
  1. ターム・配置モード（Pack / Spread）を選択する
  2. 自動生成を実行する
  3. 各カリキュラムが制約を満たすスロットに配置される
  4. 配置結果をグリッド表示で確認する
- **代替フロー**:
  - 配置不可のカリキュラムが存在する場合、警告を表示する

### UC-M1S-02: コマ入れ替え

- **アクター**: 教務管理者
- **事前条件**: 自動生成済みの時間割が存在する
- **主フロー**:
  1. 配置済みのコマをクリックする
  2. 入れ替え可能な先のコマがハイライト表示される
  3. 入れ替え先を選択して確定する
  4. 影響を受ける他のコマが自動的に再配置される
- **制約**: 入れ替えの連鎖は最大2段階まで

### UC-M1S-03: 時間割の確定

- **アクター**: 教務管理者
- **主フロー**:
  1. 配置結果を確認する
  2. 確定操作を行う
  3. M2（データ統合）へエクスポートされる

## 配置アルゴリズム

### DP（動的計画法）による配置手順

1. 各カリキュラムの候補スロット数を計算する
2. 候補が少ない順（MRV: Minimum Remaining Values）にソートする
3. 各カリキュラムを順番に配置し、制約伝播で矛盾を検出する
4. バックトラックで配置不可時に別候補を試行する

### 制約条件

- 講師の出講可能スロット内であること
- 同一講師が同一コマに複数授業を持たないこと
- 同一教室が同一コマに重複使用されないこと
- カリキュラムの必要コマ数を満たすこと

### 配置モード

| モード | 戦略 | ユースケース |
|--------|------|-------------|
| Pack（詰め込み） | 制約の厳しい科目から優先配置。早い曜日・コマに密集 | 教室稼働率最大化、省エネ |
| Spread（分散） | 曜日をまたいで均等配置。ラウンドロビン方式 | 学生の負荷分散、空きコマ確保 |

## 入れ替え候補ハイライト

### 入れ替え対象の条件

- 入れ替え先候補のコマが空欄、または入れ替え可能な授業であること
- 対象コマに対応可能な教室が存在すること
- 講師の空き時間と矛盾しないこと

### 候補数による色分け

| 候補数 | 色 | 意味 |
|--------|-----|------|
| 3コマ以下 | 灰色 (#6E7681) | 候補少 — 入れ替え困難 |
| 4〜14コマ | オレンジ (#D29922) | 候補中 — 慎重に選択 |
| 15コマ以上 | 緑 (#3FB950) | 候補多 — 柔軟性あり |

## 無限ループ防止

入れ替え操作時の再帰的再計算による無限ループを防止する。

- 配置時の候補が15コマ以上の講師の授業は再計算対象外とする
- 入れ替えの連鎖は最大2段階まで（直接入れ替え + 影響先の再配置）
- 各入れ替え操作に Transaction ID を付与し、同一 Transaction 内の再入を禁止する

## エッジケース

- すべてのカリキュラムを配置できない場合、未配置リストを返す
- 講師が1名しかいない科目は最優先で配置される（MRV）
- 教室が不足する場合、roomId を null として仮配置する
