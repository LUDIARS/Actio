# M1: 施設予約 設計仕様

## 概要

教室・施設の予約を管理するサブモジュール。予約プラグインアーキテクチャにより、共通のインターフェースで予約機能を提供する。予約作成時にカレンダー予定として自動登録され、キャンセル時に連動削除される。

## ユースケース

### UC-M1F-01: 施設予約の作成

- **アクター**: 認証済みユーザー
- **事前条件**: グループに所属していること
- **主フロー**:
  1. 予約タイトル・日時・教室・参加者を入力する
  2. 教室の空き状況を確認する
  3. 予約を確定する
  4. カレンダーに予定として自動登録される
- **代替フロー**:
  - 教室が既に使用中の場合、エラーを返す

### UC-M1F-02: 施設予約のキャンセル

- **アクター**: 予約の作成者
- **主フロー**:
  1. 予約を選択してキャンセルする
  2. 関連するカレンダー予定が連動削除される

### UC-M1F-03: 施設予約の一覧表示

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. 教室別・日付別のスケジュールを表示する
  2. 空き状況をグリッド表示で確認する

### UC-M1F-04: 施設の空き状況照会

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. 日付・コマ範囲を指定する
  2. 利用可能な教室の一覧を取得する

## 予約プラグインアーキテクチャ

施設予約は ReservationPlugin インターフェースを実装する。

### プラグイン情報

| 項目 | 値 |
|------|----|
| プラグインID | facility |
| 表示名 | 施設予約 |
| アイコン | Building2 (Lucide) |
| APIベースパス | /api/m1/facility-booking |
| フロントエンドパス | /facility-booking |

### 共通操作

| 操作 | エンドポイント | メソッド |
|------|--------------|----------|
| 予約一覧取得 | /reservations | GET |
| 予約作成 | /reservations | POST |
| 予約キャンセル | /reservations/:id | DELETE |

## カレンダー連携

- 予約作成時: personalEvent レコードとして自動登録される
- 予約キャンセル時: 関連する personalEvent が連動削除される
- カレンダー上では eventType: "school_event" として表示される

## 競合制御

- 楽観的ロック: 予約確定時にバージョンチェック、競合時は後発を拒否
- 同一教室・同一コマへの同時予約を防止する

## 公開ルール

予約情報は公開情報として扱う。

| 項目 | 公開 |
|------|------|
| 予約タイトル | 公開 |
| 曜日・コマ | 公開 |
| 教室 | 公開 |
| グループ名 | 公開 |
| 参加者名 | 公開 |
| メモ | 公開 |
| 各メンバーの授業情報 | 非公開 |
| 各メンバーの個人予定 | 非公開 |
