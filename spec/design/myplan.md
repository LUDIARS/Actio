# MyPlan（マイプラン）設計仕様

## 概要

週間ルーティーンを定義し、個人予定を自動生成するモジュール。基本パターンと特別パターンを持ち、特別パターンが優先される。

## ユースケース

### UC-MP-01: マイプラン作成

- **アクター**: 認証済みユーザー
- **主フロー**:
  1. プラン名・パターンタイプ（basic / special）を設定する
  2. 週間スケジュールを曜日ごとに登録する（各エントリに開始時刻・終了時刻・タイトルを設定）
  3. 適用期間（開始日・終了日）を任意で設定する
  4. プランを保存する
- **事後条件**: 対応する個人予定が自動生成される

### UC-MP-02: マイプラン編集

- **アクター**: プランの所有者
- **主フロー**:
  1. 既存プランの内容を変更する
  2. 関連する個人予定が再生成される

### UC-MP-03: マイプラン有効化 / 無効化

- **アクター**: プランの所有者
- **主フロー**:
  1. プランの有効/無効を切り替える
  2. 無効化すると関連する個人予定が非表示になる

### UC-MP-04: グループ向けマイプラン

- **アクター**: グループ管理者
- **主フロー**:
  1. グループIDを指定してマイプランを作成する
  2. グループの空き計算に反映される

## パターン優先度

- basic（基本パターン）: デフォルトの週間スケジュール
- special（特別パターン）: 特定期間に適用される優先スケジュール
- 同一期間に複数パターンがある場合、priority 値が大きいものを優先
- 適用期間が設定されている場合、期間外は無効扱い

## 週間スケジュール形式

```
{
  "0": [{ "startTime": "09:30", "endTime": "12:30", "title": "授業" }],
  "1": [{ "startTime": "13:30", "endTime": "15:30", "title": "自習" }],
  ...
}
```

- キー: 曜日インデックス（0=月〜6=日）
- 値: その曜日のスケジュール配列

## Plan（レガシー）との関係

- plans テーブルはコマベースの繰り返し予定生成に使用
- myPlans テーブルは時間ベースの週間ルーティーンとして導入
- 両方を併用可能だが、myPlans が推奨
