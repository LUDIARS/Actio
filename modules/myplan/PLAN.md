# マイプランモジュール計画書

## 概要

ユーザの週間ルーティーンを管理するコアモジュール。曜日ごとのスケジュールパターンを定義し、personalEvent として自動登録する。グループ単位のプランもサポート。

---

## モジュール構成

```
modules/myplan/
├── PLAN.md                     # 本設計書
└── routes.ts                   # マイプラン API ルート
```

---

## 1. データモデル

### 1.1 マイプラン (myPlans)

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string | 主キー |
| name | string | プラン名 |
| userId | string | 所有ユーザ |
| groupId | string? | グループID (グループプランの場合) |
| patternType | string | パターン種別 |
| weeklySchedule | string | 曜日別スケジュール (JSON) |
| validFrom | string? | 有効開始日 |
| validUntil | string? | 有効終了日 |
| isActive | boolean | 有効/無効 |

### 1.2 weeklySchedule 形式

```json
{
  "月": [
    { "startTime": "09:30", "endTime": "10:30", "title": "朝礼" },
    { "startTime": "13:00", "endTime": "14:00", "title": "定例MTG" }
  ],
  "火": [ ... ]
}
```

---

## 2. API エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/` | マイプラン一覧 |
| POST | `/` | マイプラン作成 |
| PUT | `/:id` | マイプラン更新 |
| DELETE | `/:id` | マイプラン削除 |
| POST | `/:id/generate` | プランから personalEvent を生成 |

---

## 3. スケジュール生成フロー

```
[マイプラン定義]
    │
    ▼
┌──────────────────────┐
│  既存プラン予定の削除   │  同じプランから生成された既存 personalEvent を削除
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  weeklySchedule 展開  │  各曜日のスロットを personalEvent に変換
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  コンフリクトチェック   │  同時間帯の既存予定との重複を確認
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  personalEvent 一括登録 │  カレンダーに反映
└──────────────────────┘
```
